    Towne Park Forecasting - Actuals Display User Process for Account Managers

    Process Overview

    This document provides Account Managers with comprehensive guidance on interpreting and working with actuals data displayed across different forecasting tabs. The process covers understanding display methodologies, analyzing variance, and making informed decisions based on actual vs. forecast performance.

    Prerequisites

    Required Access

    • Forecasting system access with Account Manager permissions
    • Access to customer sites under management
    • EDW data access for actuals information

    Required Knowledge

    • Understanding of forecasting fundamentals
    • Familiarity with customer site operations
    • Basic variance analysis concepts

    Data Requirements

    • Current month actuals data availability
    • Historical forecast/budget data
    • Employee scheduling data (for payroll analysis)

    Process Steps

    Step 1: Accessing Actuals Data

    Step 1.1: System Login and Navigation

    Action: Log into the Towne Park Forecasting system System Response: Dashboard displays with available customer sites Decision Points: Select appropriate customer site for analysis Validation: Verify correct site selection and data access permissions Error Handling: Contact system administrator if access issues occur Tips: Bookmark frequently accessed sites for quick navigation

    Step 1.2: Tab Selection and Data Verification

    Action: Navigate to desired forecasting tab (Rates, Other Expenses, Statistics, or Payroll) System Response: Tab loads with current data display format Decision Points: Determine if actuals data is available for analysis period Validation: Check data freshness indicators and last update timestamps Error Handling: If actuals data is missing, verify data refresh schedule Tips: Start with Statistics tab for quick performance overview

    Step 2: Rates Tab Analysis

    Step 2.1: Understanding Rates Display Format

    Action: Review rates data in second-row display format System Response: Displays forecast/budget in top row (blue), actuals in bottom row (black) Decision Points: Identify significant variances between forecast and actual rates Validation: Verify rate calculations align with contract terms Error Handling: Flag unusual rate variances for investigation Tips: Focus on high-volume rate categories for maximum impact analysis

    Step 2.2: Variance Analysis Process

    Action: Calculate percentage variance between forecast and actual rates System Response: System may provide automated variance calculations Decision Points: Determine if variances require action or explanation Validation: Cross-reference with market conditions and operational changes Error Handling: Document unexplained variances for escalation Tips: Use historical trends to contextualize current variances

    Step 2.3: Rate Performance Documentation

    Action: Document significant findings and recommended actions System Response: System may provide note-taking or annotation features Decision Points: Determine communication needs for stakeholders Validation: Ensure documentation includes supporting data and rationale Error Handling: Save work frequently to prevent data loss Tips: Use standardized templates for consistent reporting

    Step 3: Other Expenses Analysis

    Step 3.1: Current Month Focus

    Action: Review current month other expenses with split display System Response: Shows forecast/budget and actuals for current month only Decision Points: Assess current month expense performance against budget Validation: Verify expense categorization and allocation accuracy Error Handling: Investigate unexpected expense categories or amounts Tips: Pay special attention to controllable vs. non-controllable expenses

    Step 3.2: Expense Variance Investigation

    Action: Analyze variances between budgeted and actual expenses System Response: Display variance amounts and percentages Decision Points: Determine if variances are one-time or trending Validation: Correlate expense variances with operational activities Error Handling: Flag recurring negative variances for management attention Tips: Consider seasonal factors and timing differences in analysis

    Step 3.3: Historical Context Review

    Action: Review historical months (forecast-only display) for trends System Response: Shows forecast/budget data for previous months Decision Points: Identify patterns in expense management Validation: Compare historical accuracy of expense forecasting Error Handling: Note any data inconsistencies for IT support Tips: Use historical data to improve future forecasting accuracy

    Step 4: Statistics Tab Analysis

    Step 4.1: Actuals-First Interpretation

    Action: Review statistics with actuals replacing forecast values System Response: Displays actual performance metrics where available Decision Points: Assess volume performance against expectations Validation: Verify statistics align with operational reports Error Handling: Investigate discrepancies between system and operational data Tips: Focus on key performance indicators most relevant to site success

    Step 4.2: Performance Trend Analysis

    Action: Analyze actual performance trends over time System Response: May provide trending charts or historical comparisons Decision Points: Identify improving or declining performance areas Validation: Correlate performance trends with operational initiatives Error Handling: Document unusual performance patterns for investigation Tips: Consider external factors (weather, events, market conditions)

    Step 4.3: Forecast Accuracy Assessment

    Action: Compare actual results to original forecasts (when both available) System Response: System may highlight forecast accuracy metrics Decision Points: Evaluate forecasting methodology effectiveness Validation: Assess consistency of forecasting accuracy across metrics Error Handling: Note systematic forecasting biases for methodology review Tips: Use accuracy insights to improve future forecasting processes

    Step 5: Payroll Tab Analysis

    Step 5.1: Scheduled Hours Review

    Action: Review scheduled hours data from Legion integration System Response: Displays scheduled hours with calculated costs Decision Points: Assess labor scheduling efficiency and cost implications Validation: Verify scheduled hours align with operational needs Error Handling: Investigate scheduling discrepancies with site management Tips: Consider both labor cost and service level implications

    Step 5.2: Labor Cost Analysis

    Action: Analyze labor costs based on scheduled hours and rates System Response: Shows calculated labor costs with rate applications Decision Points: Evaluate labor cost efficiency and budget adherence Validation: Cross-reference with HRIS data for rate accuracy Error Handling: Flag rate discrepancies for HR investigation Tips: Monitor overtime trends and scheduling optimization opportunities

    Step 5.3: Workforce Planning Integration

    Action: Use payroll data for workforce planning decisions System Response: Provides data foundation for staffing decisions Decision Points: Determine staffing adjustments based on performance data Validation: Ensure staffing decisions align with service level requirements Error Handling: Escalate staffing concerns to district management Tips: Balance labor cost optimization with customer service quality

    Step 6: Cross-Tab Analysis and Decision Making

    Step 6.1: Integrated Performance Review

    Action: Synthesize insights from all tabs for comprehensive site assessment System Response: User must manually integrate insights across tabs Decision Points: Identify key performance drivers and improvement opportunities Validation: Ensure analysis considers all relevant performance dimensions Error Handling: Seek additional data if analysis reveals gaps Tips: Use structured analysis framework for consistent evaluation

    Step 6.2: Action Plan Development

    Action: Develop specific action plans based on actuals analysis System Response: System may provide action tracking or follow-up features Decision Points: Prioritize actions based on impact and feasibility Validation: Ensure action plans are specific, measurable, and time-bound Error Handling: Revise plans if initial approaches prove ineffective Tips: Involve site management in action plan development and execution

    Step 6.3: Stakeholder Communication

    Action: Communicate findings and recommendations to appropriate stakeholders System Response: System may provide reporting or export capabilities Decision Points: Determine appropriate communication channels and frequency Validation: Ensure communications are clear, accurate, and actionable Error Handling: Follow up if communications don't result in expected actions Tips: Tailor communication style and detail level to audience needs

    Alternative Flows

    Missing Actuals Data Scenario

    Trigger: Actuals data not available for expected time period Process: 1. Verify data refresh schedule and timing 2. Contact IT support if data is overdue 3. Use forecast data with appropriate caveats 4. Document data availability issues for trend tracking

    Data Quality Issues Scenario

    Trigger: Actuals data appears inconsistent or incorrect Process: 1. Cross-reference with alternative data sources 2. Document specific data quality concerns 3. Escalate to data management team 4. Use alternative analysis methods if necessary

    System Performance Issues Scenario

    Trigger: System response time or availability problems Process: 1. Document system performance issues 2. Use cached or exported data if available 3. Contact IT support for system status 4. Adjust analysis timeline if necessary

    • Verify data accuracy and completeness

    • Ensure analysis considers all relevant factors
    • Confirm recommendations are actionable

    Documentation Standards

    • Use standardized analysis templates
    • Include supporting data and rationale
    • Maintain version control for analysis updates
    • Archive analysis for historical reference

    Troubleshooting Guide

    Common Issues and Resolutions

    Issue: Actuals data not displaying Resolution: Check data refresh schedule, verify permissions, contact IT support

    Issue: Variance calculations appear incorrect Resolution: Verify source data accuracy, check calculation methodology, escalate if needed

    Issue: System performance slow Resolution: Clear browser cache, try different browser, contact IT support

    Issue: Unable to export analysis results Resolution: Check export permissions, try alternative export formats, contact system administrator
